# PKU Academic Classic — HTML Slides Maker

Fixed 1920×1080 canvas, 10 color skins + DIY custom skin for academic HTML presentations.

**Activate:** `/hep-frontend-slides` or "make slides" / "academic presentation"

---

## 🚨⛔ HARD-STOP — Three Iron Rules (violation = everything void, redo from scratch)

> ⚠️ **The following rules are absolutely inviolable. If any rule is triggered, all current output is considered invalid and must be redone from scratch. No exceptions, no workarounds.**

> 🔴 **No temporary scripts.** Creating `/tmp/*.py`, `/tmp/*.sh` or any temporary script to assemble HTML is strictly forbidden. The only scripts allowed to execute are the repository's `init-slides.py`, `bundle-html.py`, `renumber-slides.py`. Violation = **entire HTML output void**.

> 🔴 **No auto-bundling.** `bundle-html.py` **may only be run when the user explicitly says** `bundle` / `deliver`. After content injection, you must ask "Ready to bundle?" and **stop and wait for reply**. Bundling without permission = **output rejected**.

> 🔴 **No hand-written HTML skeleton.** CSS / JS / Logo / Header / Footer are all auto-generated by `init-slides.py` + empty template. AI work is **limited to** injecting content into placeholder pages. Hand-writing skeleton = **file immediately discarded**.

## Attachment Path Convention

After the user specifies `{path}/{name}.html`, automatically create:

```
{path}/
├── {name}.html
└── attachment_{name}_html/
    └── Figures/
        ├── S1/    ← Images for Slide 1
        ├── S4/    ← Images for Slide 4 (no dir for slides without images)
        ├── S5/
        └── ...
```

**Rules:**
1. After `init-slides.py` scaffold, `mkdir -p attachment_{name}_html/Figures`
2. Tell the user: **"Please place images into `Figures/S{N}/` folders by slide number"**
3. HTML image paths must be: `src="attachment_{name}_html/Figures/S{N}/plot.png"`
4. After adding/deleting slides, run `renumber-slides.py` to sync directory renaming

## Step 2: Fine Tuning

> Fine Tuning must occur before `bundle-html.py`. Editing after bundle = modifying base64.

**Trigger:** "fine tuning" / "adjust" / "fix slide X"

**⚠️ Mandatory Rule (Pre-emptive Fine-Tuning Exposure):**
When generating HTML content (especially injecting placeholders or new slides), **you must explicitly expose key layout parameters via `style="..."`** — don't let the user guess how to adjust positioning.
- **Bullet List & Boxes:** Must define `margin-top`, `left` (if offset needed), `font-size`. Example: `<ul class="bullet-list" style="margin-top: 20px; font-size: 26px;">`.
- **Tables & Captions:** Must define `margin-top`, `font-size`. Example: `<div class="caption" style="font-size: 24px; margin-bottom: 15px;">`.
- **Plots/Images:** Must expose `margin-top`, `--h` (height) on `.fig` or outer container.
- **Terminal & Code Blocks:** Must expose `margin-top` + `padding-left` on `.slide-content`; expose `margin-top`, `margin-left`, `width` on each `.mac-terminal`; expose `gap`, `margin-top` on `.two-col`.
  - **📁 Icon (HARNESS):** All `.mac-terminal-title` text **must start with `📁 `**, no exceptions. This includes typewriter terminals, static code displays, multi-language syntax examples, etc. Format examples: `📁 ky230 — -zsh — 80x24`, `📁 python3 — plot_limits.py`, `📁 C++ — Producer.cc`.
  - **CxR Size Annotation:** When a terminal title contains a shell identifier (e.g. `-zsh`, `-bash`), append `CxR` size annotation at the end (e.g. `125x30`). This value **must be precisely calculated from the terminal body's actual pixel dimensions**, never hardcoded: `C = floor(body_width_px / char_width_px)`, `R = floor(body_height_px / line_height_px)`. Where `char_width` and `line_height` depend on `.mac-terminal-code`'s `font-size` and `line-height`. If undeterminable at build time, use reasonable defaults with a comment annotation.
- **Element Labels:** When a page contains **2+ independent visual elements** (e.g. multiple terminals, left-right boxes, multi-figure areas), **each element must be preceded by a `<!-- [ELEMENT: description] -->` comment** indicating which component the code block corresponds to, for easy user navigation.

For fine-tuning, refer directly to the specs in `reference/FINE_TUNING.md`, and add `<!-- FINE TUNING: ... -->` comment blocks in the code for user self-modification.

**Protocol:**
1. **Receive feedback** — user specifies per-page changes
2. **Locate** — `grep_search` for `<!-- [Slide N] -->`, `view_file` for context
3. **Execute** — `replace_file_content` for precise replacement, following `PKU_ACADEMIC_CLASSIC.md`
4. **Structural changes** (add/delete slides) — after completion, run:
   ```bash
   python3 {{FRONTEND_SLIDES_REPO_PATH}}/scripts/renumber-slides.py <input.html>
   ```
   Auto-renumbers `<!-- [Slide N] -->` markers + renames `Figures/S*/` directories + updates `src` paths.
   Add `--dry-run` to preview changes first.

| Category | Requires re-bundle? |
|----------|:---:|
| Font size/color/spacing/alignment | ❌ |
| Text/bullet/data corrections | ❌ |
| Box class switching | ❌ |
| Add/delete slides | ❌ |
| **New local image references** | ✅ |

**Parameter quick reference:** Read `reference/FINE_TUNING.md`

**Exit Fine Tuning:** When user says `bundle` / `deliver`, run `bundle-html.py`.

## Step 3: Delivery

### 3.1 Bundle

```bash
python3 {{FRONTEND_SLIDES_REPO_PATH}}/scripts/bundle-html.py <input.html>
# Output: <input>_bundle.html (does not overwrite original)
```

### 3.2 Export PDF (optional)

```bash
bash {{FRONTEND_SLIDES_REPO_PATH}}/scripts/export-pdf.sh <path.html> [output.pdf] [--dpr N] [--compact]
```

| Flag | Default | Description |
|------|---------|-------------|
| *(positional 1)* | — | Input HTML path (required) |
| `[output.pdf]` | `<input>_export.pdf` | Output PDF path |
| `--dpr N` | `3` | Device Pixel Ratio — controls screenshot resolution |
| `--compact` | off | Use 1280×720 viewport (50-70% smaller files) |

| `--dpr` | Resolution | File Size (18 slides) | Use Case |
|---------|-----------|----------------------|----------|
| `1` | 1920×1080 | ~12MB | Daily sharing |
| `2` | 3840×2160 | ~24MB | Print |
| `3` (default) | 5760×3240 | ~48MB | Archive |
| `1 --compact` | 1280×720 | ~6MB | Quick preview |

> ⚠️ **Prerequisites:** Node.js + npm (Playwright auto-installs on first run). PDF = static screenshots, no animations.
